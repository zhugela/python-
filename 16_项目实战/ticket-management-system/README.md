# 🚄 车票管理系统

Python程序设计课程设计项目 - 车票管理系统

## 项目简介

一个基于Python的车票管理系统，支持车票的增删改查、购票、日志审计等功能，采用JSON文件进行数据持久化存储。

## 功能模块

### 🛠️ 后台数据管控（队员A）
- **添加车票**：录入新车次信息
- **修改车票**：修改车次信息，自动记录修改时间戳
- **删除车票**：假删除（移入回收站），可恢复
- **回收站管理**：查看、恢复已删除的车票
- **文件读写**：JSON格式存储，load_data() / save_data() 封装

### 🎫 前台核心业务（队员B）
- **精确查询**：按车次、出发地、目的地精确匹配
- **模糊查询**：使用 `in` 关键字进行模糊匹配
- **购票功能**：
  - 余票判断（if 余票 >= 购买数量）
  - 自动扣减库存
  - 生成带乘客姓名+身份证的购票记录
  - 自动计算总价

### 📝 日志审计（特色加分项）
- 所有操作（增删改购票）均记录到 log.txt
- 操作可追溯，体现工程化思维

## 文件结构

```
ticket-management-system/
├── main.py              # 主程序入口
├── ticket.py            # 数据模型（Ticket, BookingRecord）
├── data_manager.py      # 后台数据管理（增删改查、JSON读写）
├── booking.py           # 前台业务（查询、购票）
├── logger.py            # 日志审计
├── requirements.txt     # 依赖列表
├── README.md            # 项目说明
├── data/                # 数据目录（自动创建）
│   ├── tickets.json     # 车票数据
│   └── booking_records.json  # 购票记录
└── logs/                # 日志目录（自动创建）
    └── log.txt          # 操作日志
```

## 运行方式

```bash
python main.py
```

## 技术要点

- **数据持久化**：JSON 文件存储
- **假删除机制**：is_deleted 标记 + deleted_at 时间戳
- **修改标记**：updated_at 时间戳自动更新
- **模糊查询**：Python `in` 关键字实现
- **日志审计**：with open 追加写入模式
